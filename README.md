# ArkLive Platform – Repository Scope

## Overview

ArkLive is a **token-aware identity, reputation, and engagement platform** designed to support communities, projects, and collectives within a broader metaverse-oriented ecosystem.

This repository contains the **core application logic, user systems, and mechanics** that power ArkLive’s account infrastructure, rank progression, rewards, and on-chain awareness—without requiring mandatory wallet connections.

ArkLive is built to function as a **long-term digital Ark**: a place where users establish identity, earn reputation, contribute to systems, and participate in evolving economies and social structures.

---

## Core Purpose

The ArkLive platform exists to:

* Establish **persistent user identity** beyond wallets alone
* Track **contribution, participation, and alignment**
* Reward engagement through **rank progression and points**
* Integrate **on-chain token awareness** without forcing custody
* Enable **projects, collectives, and communities** to form inside a shared framework

It is not a single-purpose dApp, but a **foundational layer** for future experiences, tooling, and metaverse access.

---

## Key System Domains

### 1. Account & Identity Layer

* Email-based account creation with one-step verification
* Optional wallet registration (non-mandatory)
* Username / handle system
* Profile images and metadata
* Progressive profile completion flow

Designed to lower friction while maintaining a verifiable user state.

---

### 2. Rank & Reputation System

ArkLive uses a structured rank hierarchy to reflect user alignment, contribution, and progression:

**Citizen → Patriot → Cadet → Specialist → Vanguard → Architect → Arkadian**

Each rank represents:

* Increased trust
* Expanded access
* Higher reputation weight
* Eligibility for future rewards and privileges

Rank logic is designed to be **extensible**, not static.

---

### 3. Engagement & Progress Mechanics

* Quest participation and point accumulation
* Weekly status updates
* Achievement tracking with tooltips
* User “battery” / energy system representing activity and momentum

These mechanics form an **engagement loop** without gamifying users irresponsibly.

---

### 4. Token-Aware Logic ($ARDY)

The platform includes **read-only token awareness**, such as:

* Periodic balance snapshots
* Detection of balance increases, decreases, or stability
* Historical snapshot records

This allows reward logic and reputation signals **without requiring wallet connection or custody**, respecting user privacy and whale preferences.

---

### 5. Network & Association Intelligence

During onboarding or profile completion, users may optionally identify as:

* Project founders
* Collective founders
* Team members
* Community participants

This data enables:

* Automatic association
* Future group-based quests
* Collective rewards
* Ecosystem mapping

All associations are designed to be **explicit, user-declared, and revocable**.

---

### 6. Security & Access Model

* One-factor email authentication (code-based)
* No passwords stored by default
* Progressive security model designed to evolve over time

Security choices prioritize **accessibility first, escalation later**.

---

## Design Principles

* **Optionality over force**
  Wallets, social participation, and depth are opt-in.

* **Transparency without exploitation**
  Source-visible, but protected against hostile commercialization.

* **Reputation before speculation**
  Contribution and alignment are valued more than raw balance.

* **Composable future**
  Systems are designed to support future modules, worlds, and economies.
